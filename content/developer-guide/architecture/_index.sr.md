LibreSCRS се састоји од два главна пројекта који заједно читају, обрађују и приказују податке са смарт картица.

## Компоненте

### LibreMiddleware (LGPL-2.1)

Колекција статичких C++20 библиотека за комуникацију са смарт картицама, без зависности од Qt-а. Обрађује све од APDU команди ниског нивоа до екстракције података са картице.

| Библиотека | Намена |
|---|---|
| `smartcard` | PCSCConnection, APDU команда/одговор, TLV и BER-TLV парсирање (ISO 7816-4) |
| `eidcard` | API за српску eID картицу са имплементацијама читача (Apollo 2008, Gemalto 2014+, Foreigner IF2020) |
| `vehiclecard` | API за саобраћајну дозволу |
| `healthcard` | Картица здравственог осигурања (РФЗО) |
| `pkscard` | ПКС картица квалификованог електронског потписа (Привредна комора) |
| `cardedge` | Генерички CardEdge/PKCS#15 аплет — PIN операције, потписивање, откривање сертификата |
| `emrtd` | eMRTD комуникација са е-пасошем — читање група података, парсирање MRZ-а |
| `emrtd-crypto` | eMRTD криптографија — BAC, PACE (ECDH-GM), Secure Messaging |
| `pkcs11` | PKCS#11 дељена библиотека (`librescrs-pkcs11`) |
| `*-plugin` | Додаци за картице (`.so`): eidcard, vehiclecard, healthcard, pkscard, opensc, emrtd |

### LibreCelik (GPL-3.0)

Qt6 десктоп GUI апликација за приказ података са картица. Чист презентацијски слој са сопственим системом додатака за виџете специфичне за картице.

| Модул | Намена |
|---|---|
| `smartcard` | SmartCardScanner — детекција убацивања/вађења картице кроз PC/SC |
| `plugin` | CardWidgetPlugin интерфејс и CardWidgetPluginRegistry (QPluginLoader) |
| `asynccardreader` | Генерички асинхрони читач картица кроз middleware CardPlugin ланац |
| `document` | Дељени PKI кориснички интерфејс (TokenSection), дијалог за промену PIN-а, штампање |
| `certificate` | Прегледач X.509 сертификата (модел стабла + дијалог) |
| `plugins/` | GUI додаци (rseid, vehicle, health, pks) као Qt MODULE `.so` датотеке |

LibreCelik преузима LibreMiddleware преко CMake `FetchContent`. За локални развој можете усмерити на локалну копију.

## Архитектура додатака

Цео систем је изграђен око додатака. Постоје два независна слоја — **middleware додаци** за комуникацију са картицом, **GUI додаци** за приказ. Повезани су кроз `CardData`, универзални модел података који сваки middleware додатак производи, а сваки GUI додатак конзумира.

### Middleware додаци (CardPlugin)

Middleware додатак је дељена библиотека (`.so` / `.dylib`) коју `CardPluginRegistry` учитава преко `dlopen` у време извршавања. Сваки додатак имплементира две кључне методе:

- **`probe(PCSCConnection&)`** — детектује да ли овај додатак подржава убачену картицу. Типично шаље SELECT команде за познате AID-ове и враћа оцену поузданости. Регистар позива `probe()` на сваком регистрованом додатку и гради рангирану листу кандидата.
- **`readCard(PCSCConnection&)`** — екстрахује све податке са картице. Шаље APDU команде, парсира TLV/BER-TLV одговоре и враћа `CardData` објекат са типизираним групама поља (лични подаци, подаци документа, фотографије, сертификати).

**Ланац резервних опција:** Ако `readCard()` најбоље рангираног додатка не успе, аутоматски се покушава следећи кандидат. OpenSC додатак служи као генеричка резервна опција — делегира PKI операције (сертификати, потписивање) OpenSC-у за сваку картицу коју нема нативно подржану.

**Додавање новог типа картице:** Напишите класу која наслеђује `CardPlugin`, имплементирајте `probe()` и `readCard()`, компајлирајте као дељену библиотеку и поставите је у директоријум за додатке. Регистар је аутоматски открива при следећем покретању.

### GUI додаци (CardWidgetPlugin)

GUI додатак је Qt дељена библиотека коју `CardWidgetPluginRegistry` учитава преко `QPluginLoader`. Сваки додатак имплементира:

- **`cardTypes()`** — враћа листу типова картица које овај додатак може да прикаже (мора да одговара ономе што middleware додатак поставља у `CardData`).
- **`createWidget(CardData)`** — гради и враћа Qt виџет који приказује податке са картице. Пуна контрола над распоредом — текстуална поља, фотографије, сертификати, шта год картица садржи.

**Додавање новог приказа картице:** Напишите класу која наслеђује `CardWidgetPlugin` и `Q_PLUGIN_METADATA`, имплементирајте `cardTypes()` и `createWidget()`, компајлирајте као Qt MODULE библиотеку.

### Како се повезују

Додавање подршке за потпуно нови тип картице захтева два додатка:

1. **Middleware додатак** — зна како да комуницира са картицом (SELECT, READ BINARY, парсирање одговора)
2. **GUI додатак** — зна како да прикаже податке (распоред, ознаке, форматирање)

Мост између њих је `CardData` — мапа група поља, где свака група садржи парове кључ-вредност. Middleware додатак га попуњава, GUI додатак га чита. Ниједан не мора да зна за други. Поновна компилација основне апликације није потребна — само поставите `.so` датотеке.

## Ток података

Комплетан ток када се смарт картица убаци:

```
1. Картица убачена у читач

2. SmartCardScanner (PC/SC нит за прозивање)
   └─ SCardGetStatusChange детектује присуство картице
   └─ емитује cardInserted сигнал преко SmartCardReaderListener

3. Главни прозор прима сигнал, покреће откривање додатака:
   └─ CardPluginRegistry::findAllCandidates(connection)
      ├─ eidcard-plugin::probe()     → оцена: 100 (препознат AID)
      ├─ vehicle-plugin::probe()     → оцена: 0 (погрешан AID)
      ├─ emrtd-plugin::probe()       → оцена: 0 (нема eMRTD аплета)
      └─ opensc-plugin::probe()      → оцена: 50 (нађен PKCS#15)
      Резултат: [eidcard-plugin (100), opensc-plugin (50)]

4. AsyncCardReader::requestData(topCandidate)
   └─ std::async → позадинска нит
   └─ eidcard-plugin::readCard(connection)
      ├─ SELECT AID, READ BINARY лични подаци
      ├─ парсирање BER-TLV одговора
      └─ враћа CardData { type: "rs.eid", fields: {...} }

5. Резултат маршалиран назад на Qt главну нит (QMetaObject::invokeMethod)

6. CardWidgetPluginRegistry::findByCardType("rs.eid")
   └─ rseid-gui-plugin одговара

7. rseid-gui-plugin::createWidget(cardData)
   └─ гради виџет: фотографија, име, адреса, број документа, сертификати

8. Виџет приказан у главном прозору
```

## Пројектни обрасци

### Strategy

`CardReaderBase` дефинише интерфејс за комуникацију са одређеним чипом читача. Конкретне имплементације — `CardReaderApollo` (Apollo 2008 чипови) и `CardReaderGemalto` (Gemalto 2014+ чипови) — енкапсулирају разлике у APDU секвенцама и структурама датотека.

### Async

`AsyncCardReader` обавија позиве middleware додатака са `std::async` да би GUI остао одзиван. Резултати се маршалирају назад на Qt главну нит преко `QMetaObject::invokeMethod` и Qt сигнала.

### Observer

Qt сигнали и слотови пропагирају догађаје смарт картице од `SmartCardScanner`-а (нит за прозивање) преко `SmartCardReaderListener`-а (синглтон) до главног прозора и свих регистрованих слушалаца.

### Singleton

`SmartCardReaderListener::instance()` пружа јединствену тачку за диспечовање догађаја убацивања/вађења картице кроз целу апликацију.

## Стандарди

Следећи стандарди су релевантни за базу кода:

| Стандард | Примена |
|---|---|
| ISO 7816-4 | Комуникација са смарт картицом — структура APDU команде/одговора, TLV и BER-TLV кодирање |
| PC/SC | Слој за приступ читачу — детекција картице, управљање конекцијом, контрола трансакција |
| PKCS#11 | Интерфејс криптографског токена — аутентификација у прегледачу, дигитални потписи |
| PKCS#15 | Апликација криптографских информација — откривање сертификата и кључева на смарт картицама |
| ICAO 9303 | eMRTD (е-пасоши) — BAC и PACE договарање кључева, Secure Messaging, структура група података |
