LibreSCRS се састоји од два главна пројекта који заједно читају, обрађују и приказују податке са смарт картица.

## Компоненте

### LibreMiddleware (LGPL-2.1)

Колекција статичких C++20 библиотека за комуникацију са смарт картицама, без зависности од Qt-а. Обрађује све од APDU команди ниског нивоа до екстракције података са картице.

| Библиотека | Намена |
|---|---|
| `smartcard` | PCSCConnection, APDU команда/одговор, TLV и BER-TLV парсирање (ISO 7816-4) |
| `eidcard` | API за српску eID картицу са имплементацијама читача (Apollo 2008, Gemalto 2014+, Foreigner IF2020) |
| `vehiclecard` | API за саобраћајну дозволу |
| `healthcard` | Картица здравственог осигурања (РФЗО) |
| `pkscard` | ПКС картица квалификованог електронског потписа (Привредна комора) |
| `cardedge` | Генерички CardEdge/PKCS#15 аплет — PIN операције, потписивање, откривање сертификата |
| `emrtd` | eMRTD комуникација са е-пасошем — читање група података, парсирање MRZ-а |
| `emrtd-crypto` | eMRTD криптографија — BAC, PACE (ECDH-GM), Secure Messaging |
| `pkcs11` | PKCS#11 дељена библиотека (`librescrs-pkcs11`) |
| `*-plugin` | Додаци за картице (`.so`): eidcard, vehiclecard, healthcard, pkscard, opensc, emrtd |

### LibreCelik (GPL-3.0)

Qt6 десктоп GUI апликација за приказ података са картица. Чист презентацијски слој са сопственим системом додатака за виџете специфичне за картице.

| Модул | Намена |
|---|---|
| `smartcard` | SmartCardScanner — детекција убацивања/вађења картице кроз PC/SC |
| `plugin` | CardWidgetPlugin интерфејс и CardWidgetPluginRegistry (QPluginLoader) |
| `asynccardreader` | Генерички асинхрони читач картица кроз middleware CardPlugin ланац |
| `document` | Дељени PKI кориснички интерфејс (TokenSection), дијалог за промену PIN-а, штампање |
| `certificate` | Прегледач X.509 сертификата (модел стабла + дијалог) |
| `plugins/` | GUI додаци (rseid, vehicle, health, pks) као Qt MODULE `.so` датотеке |

LibreCelik преузима LibreMiddleware преко CMake `FetchContent`. За локални развој можете усмерити на локалну копију.

## Ток података

Када се смарт картица убаци, подаци пролазе кроз систем следећим редоследом:

```
Картица убачена у читач
  │
  ▼
SmartCardScanner (PC/SC нит за прозивање)
  │  детектује присуство картице преко SCardGetStatusChange
  ▼
SmartCardReaderListener (синглтон, Qt сигнали)
  │  емитује cardInserted сигнал
  ▼
Главни прозор прима сигнал
  │
  ▼
CardPluginRegistry::findAllCandidates()
  │  испробава probe() сваког middleware додатка по приоритету
  │  враћа листу додатака који препознају картицу
  ▼
AsyncCardReader::requestData()
  │  обавија middleware позив са std::async
  │  маршалира резултат назад на Qt главну нит преко сигнала
  ▼
CardPlugin::readCard() (middleware додатак)
  │  комуницира са картицом преко APDU команди
  │  парсира одговор (TLV, BER-TLV)
  │  враћа CardData са екстрахованим пољима
  ▼
CardWidgetPluginRegistry::findByCardType()
  │  мапира тип картице на GUI додатак
  ▼
CardWidgetPlugin::createWidget(CardData)
  │  гради Qt виџет за приказ података са картице
  ▼
Виџет приказан у главном прозору
```

## Архитектура додатака

Систем користи два независна регистра додатака — један за комуникацију са картицом (middleware) и један за приказ (GUI).

### Middleware додаци (CardPlugin)

Учитавају се преко `CardPluginRegistry` помоћу `dlopen` у време извршавања. Сваки додатак имплементира:

- **`probe()`** — детектује да ли овај додатак подржава убачену картицу. Враћа оцену поузданости. Регистар испробава додатке по приоритету и гради листу кандидата.
- **`readCard()`** — екстрахује све податке са картице. Враћа `CardData` објекат са типизираним пољима (текст, слике, датуми, сертификати).

Ланац резервних опција значи да ако примарни додатак не успе, аутоматски се покушава следећи кандидат. OpenSC додатак служи као генеричка резервна опција за сваку PKCS#15-компатибилну картицу.

### GUI додаци (CardWidgetPlugin)

Учитавају се преко `CardWidgetPluginRegistry` помоћу `QPluginLoader`. Сваки додатак имплементира:

- **`cardTypes()`** — листа типова картица које овај додатак може да прикаже.
- **`createWidget(CardData)`** — гради и враћа Qt виџет који приказује податке са картице.

Додавање подршке за нови тип картице значи писање две дељене библиотеке — middleware додатак за комуникацију и GUI додатак за приказ — и њихово постављање у директоријуме за додатке. Поновна компилација основне апликације није потребна.

## Пројектни обрасци

### Strategy

`CardReaderBase` дефинише интерфејс за комуникацију са одређеним чипом читача. Конкретне имплементације — `CardReaderApollo` (Apollo 2008 чипови) и `CardReaderGemalto` (Gemalto 2014+ чипови) — енкапсулирају разлике у APDU секвенцама и структурама датотека.

### Async

`AsyncCardReader` обавија позиве middleware додатака са `std::async` да би GUI остао одзиван. Резултати се маршалирају назад на Qt главну нит преко `QMetaObject::invokeMethod` и Qt сигнала.

### Observer

Qt сигнали и слотови пропагирају догађаје смарт картице од `SmartCardScanner`-а (нит за прозивање) преко `SmartCardReaderListener`-а (синглтон) до главног прозора и свих регистрованих слушалаца.

### Singleton

`SmartCardReaderListener::instance()` пружа јединствену тачку за диспечовање догађаја убацивања/вађења картице кроз целу апликацију.

## Стандарди

Следећи стандарди су релевантни за базу кода:

| Стандард | Примена |
|---|---|
| ISO 7816-4 | Комуникација са смарт картицом — структура APDU команде/одговора, TLV и BER-TLV кодирање |
| PC/SC | Слој за приступ читачу — детекција картице, управљање конекцијом, контрола трансакција |
| PKCS#11 | Интерфејс криптографског токена — аутентификација у прегледачу, дигитални потписи |
| PKCS#15 | Апликација криптографских информација — откривање сертификата и кључева на смарт картицама |
| ICAO 9303 | eMRTD (е-пасоши) — BAC и PACE договарање кључева, Secure Messaging, структура група података |
