Смарт картице са приватним кључевима могу се користити за дигиталне потписе. Ова страница покрива потписивање датотека из командне линије и помиње друге апликације које подржавају PKCS#11 потписивање. Примери испод користе српске личне картице и ПКС картице, али исти ток рада важи за сваку картицу коју подржава OpenSC или LibreSCRS PKCS#11 модул.

## Предуслови

Потребно вам је једно од следећег:

1. **OpenSC са уграђеном подршком за српске картице** — само инсталирајте OpenSC, ништа друго није потребно. *(Још увек није објављено — уграђени драјвер је одобрен и чека укључивање у следеће издање OpenSC-а.)*
2. **OpenSC + LibreSCRS екстерни драјвер** — користите ово док уграђени драјвер не буде објављен. Даје вам приступ свим OpenSC CLI алатима (`pkcs15-crypt`, `pkcs11-tool`, итд.). Погледајте [OpenSC драјвер подешавање](/sr/user-guide/cardedge-opensc-driver/).
3. **LibreSCRS PKCS#11 модул** — ради без OpenSC-а. За аутентификацију у прегледачу и потписивање преко било које PKCS#11 апликације. Погледајте [PKCS#11 подешавање](/sr/user-guide/pkcs11/).

## Потписивање из командне линије са OpenSC

### Потписивање датотеке

`pkcs15-crypt --sha-256` очекује **унапред израчунат бинарни хеш** као улаз, не сирову поруку.

```bash
# Израчунај хеш
openssl dgst -sha256 -binary /path/to/message.txt > /tmp/hash.bin

# Потпиши кључем 02 (Дигитални потпис)
pkcs15-crypt --sign --pkcs1 --sha-256 --key 02 \
    --input /tmp/hash.bin --output /tmp/sig.bin
```

### Верификација потписа

```bash
# Извуци јавни кључ из сертификата
# Напомена: pkcs15-tool враћа PEM на Linux-у, DER на macOS-у
pkcs15-tool --read-certificate 02 --output /tmp/cert.pem
openssl x509 -in /tmp/cert.pem -pubkey -noout > /tmp/pubkey.pem

# Верификуј
openssl dgst -sha256 -verify /tmp/pubkey.pem \
    -signature /tmp/sig.bin /path/to/message.txt
# Verified OK
```

### Коришћење pkcs11-tool

Можете такође потписивати користећи `pkcs11-tool` са LibreSCRS PKCS#11 модулом или OpenSC-овим модулом:

```bash
pkcs11-tool --module /usr/local/lib/librescrs-pkcs11.so \
    --sign --mechanism RSA-PKCS --id 02 \
    --input-file /tmp/hash.bin --output-file /tmp/sig.bin
```

## Аутентификација у прегледачу (еУправа)

За аутентификацију на државне сервисе као што је еУправа, подесите PKCS#11 модул у Firefox-у. Погледајте [PKCS#11 упутство за Firefox](/sr/user-guide/pkcs11/).

Када посетите сајт који захтева аутентификацију клијентским сертификатом, Firefox ће вас питати да изаберете сертификат и унесете PIN.

## Потписивање PDF докумената

PDF читачи који подржавају PKCS#11 безбедносне уређаје би требало да могу да користе вашу смарт картицу за потписивање. То укључује:

- **Okular** (KDE) — подржава PKCS#11 преко NSS-а
- **Adobe Reader** (ако је доступан на вашој платформи)

Ова функционалност није детаљно тестирана. Ако имате искуства са потписивањем PDF докумената смарт картицама, молимо [пријавите своја открића](https://github.com/LibreSCRS/LibreMiddleware/issues).

## Потписивање електронске поште (S/MIME)

Thunderbird и други клијенти електронске поште који подржавају PKCS#11 безбедносне уређаје могу да користе сертификате са ваше смарт картице за S/MIME потписивање и шифровање поште. Учитајте PKCS#11 модул у Thunderbird-у на исти начин као у Firefox-у — погледајте [PKCS#11 подешавање](/sr/user-guide/pkcs11/).

Да ли су сертификати на вашој картици погодни за S/MIME зависи од њихових екстензија за употребу кључа. Ако сте ово тестирали, молимо [поделите своје резултате](https://github.com/LibreSCRS/LibreMiddleware/issues).
